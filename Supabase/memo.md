# 振り返り用メモ
📝 認証とは
- 「ユーザーが本人であるかどうかを確認する」仕組み」
- ユーザーがログイン時にメール・パスワードを入力
- Supabaseがそれを検証し、有効なセッショントークンを発行
- トークンが有効な間は、そのユーザーは「ログイン中」として扱われる

📝 セッションとは
- Supabaseは **Cookie** ベースでセッション管理を行っている
- signInWithPassword() に成功すると、セッショントークンが自動で保存
- 現在のセッション状態は supabase.auth.getSession() で取得できる
